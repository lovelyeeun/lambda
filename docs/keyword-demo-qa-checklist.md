# 키워드 데모 QA 체크리스트

대상 키워드:
- 사무용 의자
- 청소기
- 모니터
- 토너
- A4용지
- 노트북
- 데스크
- 포스트잇
- 태블릿
- 정수기

## 목적

키워드 기반 데모 시나리오에서 누락이 반복되지 않도록, 검색 흐름과 우측 패널 동작을 같은 기준으로 점검한다.

## 사전 조건

- 시작 화면의 `키워드 기반 데모 시나리오` 칩 또는 채팅 직접 입력으로 테스트한다.
- 개발 환경에서는 브라우저 콘솔에 `[demo-scenario-check]` 경고가 없는지 먼저 확인한다.
- 조건 카드 제출 전에는 추천 결과 상품 카드가 보이지 않아야 한다.
- 조건 카드 제출 후에만 추천 결과 상품 카드가 보여야 한다.

## 공통 QA 절차

각 키워드마다 아래 순서로 확인한다.

1. 검색어 입력
- 해당 키워드로 검색했을 때 `의도 분석` → `상품 검색` → `조건 카드` 순으로 흐름이 자연스럽게 진행되는지 확인

2. 조건 카드
- 가격대 / 브랜드 / 옵션 그룹이 노출되는지 확인
- 조건을 하나 이상 조정하거나 그대로 `맞춤 추천 보기`를 눌렀을 때만 추천 결과 카드가 나타나는지 확인

3. 추천 결과 카드
- 내부 상품과 외부 상품이 함께 보이는지 확인
- 외부 상품이 있으면 가격 비교 또는 스크래핑 진행 상태가 연결되는지 확인
- AI 추천 이유가 축약형으로 보이고 `더 보기`가 동작하는지 확인

4. 상세 보기
- 1순위 상품과 후보 상품 둘 다 `상세 보기` 진입이 가능한지 확인
- 우측 패널이 열리고, 이전 상품 정보가 남지 않는지 확인
- 옵션 선택 드롭다운이 항상 보이는지 확인
- 옵션 드롭다운이 열리고 닫히며, 선택값이 정상 반영되는지 확인

5. 패널 전환
- 우측 패널을 닫았다 다시 열었을 때 stale 콘텐츠가 남지 않는지 확인
- 다른 상품 상세를 연속 클릭했을 때 옵션/설명/스펙이 현재 상품 기준으로 바뀌는지 확인

## 키워드별 기대 포인트

### 사무용 의자
- 메쉬/착석감/보증 관련 추천 문맥이 보여야 함
- 옵션 예시: 색상, 헤드레스트, 구성

### 청소기
- 흡입력/배터리/공간 적합성 문맥이 보여야 함
- 외부 상품 선택 시 스크래핑 카드가 자연스럽게 이어져야 함

### 모니터
- 4K, 포트, 업무 표준 문맥이 보여야 함
- 상세 패널에서 옵션 드롭다운이 누락되지 않아야 함

### 토너
- 호환 기종, 정품/호환/재생 문맥이 보여야 함
- 외부 상품은 리뷰/호환 정보 수집 단계가 보여야 함

### A4용지
- 재구매, 평량, 묶음 구성 문맥이 보여야 함
- 한 번에 결과 카드가 뜨지 않고 조건 카드 단계를 거쳐야 함

### 노트북
- 휴대성, 성능, 표준 스펙 문맥이 보여야 함
- 옵션 예시: 메모리/번들/크기

### 데스크
- 높이조절, 상판, 설치 편의성 문맥이 보여야 함
- 가구 카테고리처럼 우측 패널 옵션이 안정적으로 보여야 함

### 포스트잇
- 규격, 점착력, 대량구매 문맥이 보여야 함
- 사무용품 키워드라도 조건 카드와 결과 카드 흐름이 유지되어야 함

### 태블릿
- 펜 사용성, 액세서리, 휴대성 문맥이 보여야 함
- 옵션 예시: Wi-Fi/셀룰러, 커버 번들

### 정수기
- 관리 방식, 렌탈 조건, 설치 편의성 문맥이 보여야 함
- 생활용품 카테고리에서도 옵션 드롭다운이 동일 패턴으로 보여야 함

## 회귀 테스트

아래 전환도 함께 확인한다.

- `내 활동 → 채팅` 이동 시 우측 패널 잔상이 남지 않는지
- 채팅에서 상품 상세 패널을 연 뒤 다른 키워드 검색으로 넘어가도 패널 상태가 꼬이지 않는지
- 추천 결과에서 후보 상품을 눌렀을 때도 상세 패널이 정상 오픈되는지

## 운영 원칙

- 데모용 키워드를 추가하거나 바꿀 때는 먼저 `data/chat-search-scenarios.ts`에 전용 시나리오를 추가한다.
- 내부 상품은 `options`와 `aiNote`를 기본값처럼 취급한다.
- 외부 상품은 `scrapingSteps`를 반드시 포함한다.
- QA 완료 전에는 시작 화면의 데모 키워드 칩에 새 키워드를 노출하지 않는다.
