# 채팅 검색 시나리오 및 분기 구조

> 메인 채팅(`/chat`)에서 상품 검색 및 구매 플로우 테스트용.
> 이 문서의 모든 케이스는 더미 시나리오(키워드 매칭 기반)로 작동합니다.
> **구현 파일**: `components/chat/ChatContainer.tsx` — `handleSend()` 함수

---

## 1. 전체 분기 구조

사용자가 메시지를 입력하면 다음 순서로 분기됩니다. **먼저 매칭된 분기에서 처리하고 return**합니다.

```
사용자 입력
  │
  ├─ [1] 비용 분석 감지 ──→ 지출 요약 카드 + 비용 인텔리전스 유도
  │
  ├─ [2] 간식 추천 감지 ──→ 2턴 대화 (확인 → 추천)
  │
  ├─ [3] Work Item 생성 ──→ 키워드별 WI 자동 생성 (검색은 안 함, 통과)
  │
  ├─ [4] 재검색 조건 ────→ 이미 검색 결과 있을 때 정렬 변경
  │
  ├─ [5] 3-DB 시나리오 ──→ 풀 검색 애니메이션 (분석→DB스캔→추천)
  │
  ├─ [6] 정확 키워드 ────→ products에서 ID로 즉시 표시
  │
  ├─ [7] Fuzzy 검색 ────→ 상품명/브랜드/카테고리/설명 텍스트 매칭
  │     └─ 카테고리 유의어 확장 (노트북→전자기기 등)
  │
  └─ [8] 미매칭 Fallback ─→ "해당 상품을 찾지 못했습니다" + 카테고리 안내
```

---

## 2. 분기별 키워드 & 테스트 가이드

### [1] 비용 분석 — `detectCostAnalysisQuery()`

비용/지출 관련 질문을 감지해 지출 요약 카드를 표시하고 `/cost-intel` 페이지로 유도.

| 키워드 | 입력 예시 |
|---|---|
| `지출`, `비용`, `예산 분석`, `예산 얼마`, `얼마 썼` | "이번 달 지출 얼마야?" |
| `카테고리별`, `팀별 지출`, `부서별 지출` | "부서별 지출 보여줘" |
| `비용 분석`, `지출 분석`, `지출 리포트`, `지출 현황` | "지출 현황 알려줘" |
| `절감`, `절약` | "비용 절감 방법 있어?" |

---

### [2] 간식 추천 — `detectSnackQuery()`

2턴 대화 플로우: 키워드 감지 → 팀 인원/예산 확인 → 사용자 확인 → 추천 카드.

| 키워드 | 입력 예시 |
|---|---|
| `간식`, `스낵`, `다과`, `간식 추천` | "이번 달 간식 추천해줘" |
| `이번달 간식`, `팀 간식`, `간식 구매` | "팀 간식 구매하려고" |
| `간식 패키지` | "간식 패키지 보여줘" |

---

### [4] 재검색 조건 — `findResearchCondition()`

**선행 조건**: 이미 [5]번 3-DB 검색이 완료된 상태에서만 작동.

| 키워드 | 동작 |
|---|---|
| `저렴`, `싼`, `가격`, `비용`, `절감`, `예산` | 가격 낮은 순 재정렬 |
| `빠른`, `급한`, `빨리`, `배송`, `당일`, `내일` | 배송 빠른 순 재정렬 |
| `인기`, `많이`, `추천`, `좋은`, `베스트` | 구매 인기도 순 재정렬 |
| `다른`, `대안`, `그외`, `더 보여줘`, `추가` | 추가 옵션 안내 |

### [5] 3-DB 검색 시나리오 — `findScenario()` (풀 애니메이션)

가장 풍부한 검색 경험. 분석→DB 스캔→상품 추천 3단계 애니메이션 후 소싱된 상품 카드를 표시.
**현재 3개 시나리오만 구현됨.**

| 시나리오 | 키워드 | 추천 상품 수 | 후보 상품 수 |
|---|---|---|---|
| 청소기 | `청소기`, `청소` | 3개 | 3개 |
| 모니터 | `모니터`, `디스플레이` | 3개 | 3개 |
| 의자 | `의자`, `사무용의자`, `체어` | 3개 | 3개 |

> 이 시나리오에 해당하지 않으면 [6] 정확 키워드 매칭으로 넘어감.

| `데스크`, `책상`, `높이조절` | 한화 전동 데스크 |

### [7] Fuzzy 검색 — `searchProductsFuzzy()` (신규)

[6]에서 미매칭 시, 전체 상품 데이터를 대상으로 텍스트 매칭 시도.

**7a. 텍스트 매칭**: 상품의 `name`, `brand`, `category`, `description`에서 입력 텍스트 포함 여부 확인.

| 입력 예시 | 매칭 원리 | 결과 |
|---|---|---|
| "삼성" | brand: "Samsung" → 이름에 "삼성" 포함 | 갤럭시탭 |
| "LG" | name에 "LG" 포함 | LG 모니터 |
| "복사" | name에 "복사용지" 포함 | A4 복사용지 |
| "커피" | name에 "커피" 포함 | 드립백 커피 |
| "견과" | name에 "하루견과" 포함 | 우고래빗 견과 |

**7b. 카테고리 유의어 확장**: 텍스트 매칭 실패 시, 유의어 사전으로 카테고리 추론.

| 입력 | 추론 카테고리 | 결과 |
|---|---|---|
| `노트북`, `컴퓨터`, `PC` | 전자기기 | LG 모니터 + 갤럭시탭 |
| `마우스`, `키보드`, `헤드셋`, `충전기`, `스피커` | 전자기기 | LG 모니터 + 갤럭시탭 |
| `가구`, `테이블`, `선반` | 가구 | 시디즈 의자 + 한화 데스크 |
| `문구` | 사무용품 | 파인라이너 + 포스트잇 |
| `과자`, `음료`, `생수` | 간식 | 간식 상품들 |
| `세제`, `휴지`, `물티슈` | 생활용품 | 정수기 + 티포트 |

> 카테고리 매칭 시 응답: "정확한 상품은 없지만 관련 카테고리에서 N개 상품을 찾았습니다"

---

### [8] 미매칭 Fallback

위 모든 단계에서 매칭 실패 시, 카테고리 목록을 안내하는 명확한 메시지 표시.

```
"{입력}"에 해당하는 상품을 찾지 못했습니다.

현재 등록된 카테고리: 용지 · 잉크/토너 · 사무기기 · 가구 · 전자기기 · 사무용품 · 생활용품 · 간식

카테고리나 구체적인 상품명으로 다시 검색해보시겠어요?
```

## 3. 데모 추천 시나리오 (테스트 순서)

프로토타입 데모 시 아래 순서를 권장합니다.

### 시나리오 A: 풀 구매 플로우 (3-DB 검색)
1. **"사무용 의자 추천해줘"** → 3-DB 검색 애니메이션 → 3개 상품 추천
2. "좀 더 저렴한 거" → 재검색: 가격순 정렬
3. 상품 클릭 → 상세 패널 → "장바구니 담기"
4. 장바구니에서 "품의 요청"
5. 승인 → 결제 → 배송 추적

### 시나리오 B: 빠른 키워드 검색
1. **"A4 용지"** → 즉시 상품 표시
2. **"토너 필요해"** → 즉시 상품 표시
3. **"정수기 렌탈"** → 정수기 상품 표시

### 시나리오 C: Fuzzy 검색 확인
1. **"삼성"** → 갤럭시탭 (브랜드 매칭)
2. **"노트북 사고싶어"** → 카테고리 유의어로 전자기기 표시
3. **"운동화"** → 미매칭 → 카테고리 안내 메시지

### 시나리오 D: 멀티 Work Item
1. **"청소기 찾아줘"** → WI "청소기" 생성 + 3-DB 검색
2. **"A4 용지도 필요해"** → WI "A4용지" 추가 생성 + 키워드 검색
3. 우측 패널 칩 스위처로 두 WI 간 전환 확인

### 시나리오 E: 비구매 질문
1. **"이번 달 지출 얼마야?"** → 비용 분석 카드
2. **"팀 간식 추천해줘"** → 2턴 간식 추천 플로우

## 4. 조건 추가 시나리오 (검색 결과 후 후속 대화)

검색 결과가 표시된 후(`searchPhase === "results"`), 사용자가 후속 질문을 하면 아래 3가지 유형으로 분기합니다.
입력창 플레이스홀더: `"조건 추가 (예: 더 저렴한 걸로, 배송 빠른 걸로) 또는 새 검색..."`

### 4-A. 조건 변경 — 상품 리스트가 바뀜

기존 추천 상품을 **다른 기준으로 재정렬하거나 필터링**합니다. `lastScenario.products`와 `candidates`를 활용.

| 유형 | 키워드 | 동작 | AI 응답 예시 |
|------|--------|------|-------------|
| **가격 우선** | `저렴`, `싼`, `가격`, `비용`, `절감`, `예산` | 가격 낮은 순 정렬, 1순위 교체 | "가격 우선으로 재정렬했습니다." |
| **배송 우선** | `빠른`, `급한`, `빨리`, `배송`, `당일`, `내일` | 배송일 짧은 순 정렬 | "배송 속도 우선으로 재정렬했습니다." |
| **인기도 우선** | `인기`, `많이`, `추천`, `좋은`, `베스트` | 구매횟수 높은 순 정렬 | "구매 인기도 기준으로 재정렬했습니다." |
| **다른 브랜드** | `다른 브랜드`, `다른 제조사`, `브랜드 바꿔` | 후보 상품에서 현재 미포함 브랜드 교체 | "다른 브랜드 위주로 교체했습니다." |
| **추가 옵션** | `다른`, `대안`, `그외`, `더 보여줘`, `추가` | 후보 상품을 선정 목록에 합류 | "후보 상품을 추가로 포함했습니다." |

**테스트 예시:**
1. "사무용 의자 추천해줘" → 3개 추천
2. "더 저렴한 걸로" → 가격순 재정렬
3. "다른 브랜드도 보여줘" → 후보 상품에서 교체

### 4-B. 비교/판단 도움 — 상품 유지, AI 분석 메시지

현재 추천 상품은 그대로 두고, **비교 분석이나 결정 기준을 AI가 정리**해줍니다.

| 유형 | 키워드 | 동작 | AI 응답 형태 |
|------|--------|------|-------------|
| **비교 요청** | `비교`, `차이`, `뭐가 달라`, `어떤 게 나아`, `어떤 거` | 현재 추천 상품 비교표 생성 | 마크다운 표 (가격/배송/특징) |
| **결정 기준** | `결정`, `기준`, `잡아줘`, `어떻게 골라`, `고르는 법` | 상황별 추천 기준 제안 | 조건별 추천 (가성비/긴급/품질) |
| **과거 이력** | `지난번`, `전에`, `이전`, `뭘 샀`, `구매 이력` | 과거 구매 기반 추천 | 최근 구매 상품 + 재구매 제안 |

**테스트 예시:**
1. "사무용 의자 추천해줘" → 3개 추천
2. "이 중에 뭐가 나아?" → 3개 비교표 + 상황별 추천
3. "지난번엔 뭘 샀어?" → "최근 시디즈 T50 AIR 구매 이력" + 재구매 제안

### 4-C. 수량/조건 확정 — 장바구니 이행

사용자가 특정 상품과 수량을 확정하면 **장바구니에 바로 담깁니다**.

| 유형 | 키워드 | 동작 |
|------|--------|------|
| **순번 지정** | `첫 번째`, `1번`, `두 번째`, `2번`, `세 번째`, `3번` + 수량 | 해당 상품 N개 장바구니 담기 |
| **AI 추천 수락** | `추천대로`, `그걸로`, `그거`, `추천 상품` | AI 추천(isRecommended) 1개 장바구니 담기 |
| **이름 지정** | 상품 이름 일부 + 수량 | 매칭되는 상품 장바구니 담기 |

**테스트 예시:**
1. "사무용 의자 추천해줘" → 3개 추천
2. "첫 번째 걸로 5개 담아줘" → 삼성 비스포크 제트 5개 장바구니
3. 또는 "추천대로 해줘" → AI 추천 1순위 1개 장바구니

## 5. 데모 추천 시나리오 (테스트 순서)

프로토타입 데모 시 아래 순서를 권장합니다.

### 시나리오 A: 풀 구매 플로우 (3-DB 검색 + 조건 추가)
1. **"사무용 의자 추천해줘"** → 3-DB 검색 애니메이션 → 3개 상품 추천
2. **"좀 더 저렴한 거"** → 가격순 재정렬
3. **"이 중에 뭐가 나아?"** → 비교표 + 상황별 추천
4. **"첫 번째 걸로 5개"** → 장바구니 담기
5. 장바구니에서 "품의 요청"
6. 승인 → 결제 → 배송 추적

### 시나리오 B: 빠른 키워드 검색
1. **"A4 용지"** → 즉시 상품 표시
2. **"토너 필요해"** → 즉시 상품 표시
3. **"정수기 렌탈"** → 정수기 상품 표시

### 시나리오 C: Fuzzy 검색 확인
1. **"삼성"** → 갤럭시탭 (브랜드 매칭)
2. **"노트북 사고싶어"** → 카테고리 유의어로 전자기기 표시
3. **"운동화"** → 미매칭 → 카테고리 안내 메시지

### 시나리오 D: 멀티 Work Item
1. **"청소기 찾아줘"** → WI "청소기" 생성 + 3-DB 검색
2. **"A4 용지도 필요해"** → WI "A4용지" 추가 생성 + 키워드 검색
3. 우측 패널 칩 스위처로 두 WI 간 전환 확인

### 시나리오 E: 비구매 질문
1. **"이번 달 지출 얼마야?"** → 비용 분석 카드
2. **"팀 간식 추천해줘"** → 2턴 간식 추천 플로우

## 6. 알려진 제한사항

- 3-DB 풀 검색 시나리오는 **청소기/모니터/의자** 3개만 구현. 나머지는 [6] 정확 키워드 또는 [7] fuzzy 검색으로 처리.
- `products.ts`에 등록되지 않은 상품(예: 노트북, 마우스, 키보드)은 카테고리 유의어로 **유사 카테고리 상품**을 보여주지만 정확한 상품은 아님.
- 간식 추천 플로우는 2턴 고정 스크립트.
- 자연어 질문(예: "회의실에 필요한 거 다 알려줘")은 처리하지 않음.
- 조건 추가(4-A~C)는 3-DB 시나리오 검색 결과에서만 작동. 키워드/fuzzy 검색 결과에서는 미지원.
