# 설정 어시스턴트 테스트 시나리오

> 우측 사이드바 하단 **프로필 → 설정** 진입 후 좌측 채팅창에서 테스트.
> 이 문서의 모든 케이스는 더미 시나리오(키워드 매칭 기반)로 작동합니다.
> 자유 입력도 가능하지만, 매칭된 키워드 외 입력은 기본 안내 메시지로 폴백됩니다.

---

## 1. 채팅 입력 → 추론 + 우측 패널 자동 전환

채팅 어시스턴트의 핵심 기능. 입력 → AI 추론(progressive) → 우측 GUI가 해당 패널로 자동 전환 → Diff 카드로 적용/취소.

### 1-A. 예산 — 마케팅팀 월 예산 변경 ✏️ 편집 가능

| 입력 예시 | 기대 동작 |
|---|---|
| `마케팅팀 예산 500만원으로 올려줘` | • 우측 패널 → 예산 설정으로 전환<br>• 추론 4단계 순차 등장<br>• 마케팅 부서 행 자동 펼침 + 자동 스크롤<br>• Diff 카드에 **월 예산 입력 박스** 노출 (5,000,000 기본)<br>• 입력값 변경 시 연간/전사비중 미리보기 실시간 갱신<br>• 적용 → 우측 패널 마케팅 행이 새 값으로 변경 |

**편집 테스트**: Diff 카드의 "월 예산" 필드를 `7000000`으로 수정 후 적용 → 우측 패널 마케팅 행 연간 예산이 8,400만원으로 표시.

---

### 1-B. 결제수단 — BNPL 활성화

| 입력 예시 | 기대 동작 |
|---|---|
| `BNPL 연동해줘` | • 우측 → 결제수단 관리로 전환<br>• 추론 3단계<br>• Diff: "소모품 구매 BNPL: 미사용 → 사용중"<br>• 적용 → 결제수단 표의 'BNPL 소모품 구매' 행이 **사용중**으로 변경 + 자동 스크롤 |

대안 키워드: `BNPL 활성화해줘`, `후불 켜줘`

---

### 1-C. 결제수단 — 법인카드 월 한도 변경 ✏️ 편집 가능

| 입력 예시 | 기대 동작 |
|---|---|
| `법인카드 월 한도 200만원으로 올려줘` | • 우측 → 결제수단 관리로 전환<br>• Diff에 **월 한도 입력 박스** 노출 (2,000,000 기본)<br>• 적용 → 'pm-1' 행의 예산 (배정) 여부가 새 값으로 표시 |

**편집 테스트**: Diff의 "월 한도"를 `3000000`으로 수정 후 적용 → 표에 3,000,000원 표시.

대안 키워드: `카드 한도 올려줘`

---

### 1-D. 배송지 — 새 지사 추가 ✏️ 인라인 폼

| 입력 예시 | 기대 동작 |
|---|---|
| `분당 지사 배송지 추가해줘` | • 우측 → 배송지 관리로 전환<br>• 추론 3단계<br>• 채팅에 **인라인 편집 폼 카드** 등장<br>• 이름·주소는 기본값 채워짐, 수령인·연락처는 빈칸<br>• 빈칸 채우거나 "자세히 수정"으로 풀 모달 사용 가능<br>• 적용 → 우측 배송지 리스트에 새 행 추가 |

**필수 필드**: 이름, 수령인, 연락처, 주소가 모두 채워져야 적용 버튼 활성화.

---

### 1-E. 회사 정보 — 업종/업태 보완

| 입력 예시 | 기대 동작 |
|---|---|
| `회사 정보 등록해줘` | • 우측 → 회사 정보로 전환<br>• 추론 2단계 (회사명·업종 필드 순차 자동 스크롤)<br>• Diff: "업종 (미입력) → 소프트웨어 개발업", "업태 (미입력) → 정보통신업"<br>• 적용 → 회사 정보 패널의 업종·업태 필드 채워짐 |

대안 키워드: `회사 정보 확인해줘`

### 1-F. 결제권한 해제 — 영향 분석 카드

| 입력 예시 | 기대 동작 |
|---|---|
| `김지현 결제권한 해제해줘` | • 우측 → 승인 체계로 전환<br>• 추론 3단계<br>• **Impact 카드** 노출 (warning + 영향 리스트 3개)<br>• Suggestions: "박은서로 대체 지정해줘" 등 |

**연속 시나리오**: Suggestion `박은서로 대체 지정해줘` 클릭 → 결재 라인 변경 Diff 카드.

| 입력 예시 | 기대 동작 |
|---|---|
| `전체 설정 상태 알려줘` | • 추론 3단계 (전체 스캔/우선순위/리스크)<br>• 우측 패널 전환 없음 (대시보드 유지)<br>• Suggestions: "팀원부터 추가할게" 등 |

### 1-H. 팀원 — 채팅에서 직접 한 명 초대

이름·이메일·부서·역할이 한 줄에 포함되면 채팅에서 바로 초대 Diff 카드 생성.

| 입력 예시 | 기대 동작 |
|---|---|
| `최동현 donghyun@rawlabs.io 개발 구매담당 초대` | • 우측 → 팀원 관리로 전환<br>• 추론 3단계 (현황·라이선스·중복 확인)<br>• Diff 카드: 이름/이메일/부서/역할 미리보기<br>• 적용 → 팀원 패널 하단 **초대 대기** 섹션에 행 추가 + 대시보드 카드 카운트 갱신 |
| `한예진 yejin@rawlabs.io 마케팅 일반 초대` | 동일 패턴 |

**파싱 규칙**: 이메일 정규식 매칭, 한글 이름(2~4자) 첫 매칭, `경영지원/마케팅/디자인/개발` 키워드 매칭, `관리자/매니저/구매담당/일반` 키워드 매칭. 누락 시 기본값.

### 1-H'. 팀원 — 일반 안내 (모달 유도)

| 입력 예시 | 기대 동작 |
|---|---|
| `팀원 추가하고 싶어` | • 우측 → 팀원 관리로 전환<br>• "한 명만 초대하시려면…여러 명은 우측 모달" 안내<br>• Suggestions 클릭으로 1-H 시나리오 진입 가능 |

## 2. 우측 GUI 액션 → 채팅 가벼운 ack

원칙: **우측에서 한 변경은 채팅에 추론·시스템 칩을 만들지 않음.** 의미 있는 액션만 짧은 한 줄.

| 우측 액션 | 채팅 변화 | 우측 토스트 |
|---|---|---|
| input 타이핑 | 무음 | — |
| 토글 / 셀렉트 변경 | 무음 | — |
| 배송지 **저장** / 회사정보 **저장** | "저장했어요. 다른 설정도 도와드릴까요?" + suggestions | "저장되었습니다" |
| 배송지 **추가** (모달) | "추가했어요." | "추가되었습니다" |
| 배송지 **삭제** | "삭제했어요. 관련 설정에 영향이 있다면 알려드릴게요." | "삭제되었습니다" |
| 카드 **클릭** (대시보드 → 패널 진입) | 짧은 인사 + suggestions (추론 없음) | — |

## 3. 우측 GUI 자체 동작

| 화면 | 동작 |
|---|---|
| 배송지 관리 — **연필 아이콘** | 배송지 수정 모달 열림 (이미지 시안과 동일 필드 구성) |
| 배송지 관리 — **휴지통 아이콘** | 즉시 삭제 + 토스트 |
| 배송지 관리 — **기본으로** 텍스트 | 기본 배송지 변경 |
| 배송지/배송지 추가 모달 — **주소 검색** 버튼 | "예정: 우편번호·주소 검색 연동 (다음 주소 API)" 툴팁 (실제 검색 미구현) |
| 팀원 관리 — **팀원 초대** 버튼 | 탭 모달 (메일로 초대 / 엑셀로 초대) — 아래 3-A·3-B 참조 |
| 팀원 관리 — 초대 대기 행 **휴지통** | 초대 즉시 취소 + 토스트 |
| 결제수단 — **BNPL 신청** 버튼 | 신청 모달 → 신청 완료 → ⚙ 승인 처리 (프로토타입) → 승인 완료 → BNPL 추가 가능 |

### 3-A. 팀원 초대 — 메일로 초대 (다중 입력)

1. 우측 팀원 관리 → **팀원 초대** 버튼 → 모달 오픈, 기본 탭 "메일로 초대"
2. 이름/이메일/부서/역할 입력 (이름·이메일 필수)
3. **+ 행 추가** 버튼으로 여러 명 동시 입력 가능
4. 휴지통 아이콘으로 행 삭제 (마지막 1행은 삭제 불가)
5. **초대하기 (N명)** 버튼 클릭 → store에 추가 + 토스트 + 모달 닫힘
6. 패널 하단 **초대 대기** 섹션에 새 행들 표시 (Mail 아이콘)

### 3-B. 팀원 초대 — 엑셀로 초대 (일괄)

1. 모달에서 **엑셀로 초대** 탭으로 전환
2. **다운로드** 버튼 → "예정: 엑셀 템플릿 다운로드" 툴팁 (실파일 미생성)
3. **파일 선택** 버튼 → OS 파일 picker → 선택한 파일명 표시 + ✓ 업로드 완료 안내
4. 파일에 X 버튼으로 제거 가능
5. **초대하기** → 더미 파싱 결과 (최동현/한예진/정수민 3명) store 추가
6. 패널 하단 **초대 대기** 섹션에 3명 추가 (FileSpreadsheet 아이콘)

## 4. 양방향 동기화 검증 시나리오

채팅과 우측이 같은 store 를 본다는 걸 확인하는 케이스.

### 4-A. 채팅으로 변경 → 우측 즉시 반영
1. 채팅: `마케팅팀 예산 500만원으로 올려줘` → 적용
2. 우측 예산 패널의 마케팅 행이 **연 6,000만원**으로 변경 확인
3. **대시보드로 돌아가기** ("카드 목록으로") → 예산 카드의 연간 예산·사용률도 갱신 확인

### 4-B. 우측에서 변경 → 채팅 ack
1. 우측 회사 정보 패널에서 **수정** → 회사명 변경 → **저장**
2. 채팅에 "저장했어요. 다른 설정도 도와드릴까요?" 메시지 등장
3. 대시보드 카드의 회사명도 변경된 값으로 업데이트

### 4-C. 채팅 시나리오 ↔ 적용 후 강조
1. 채팅: `BNPL 연동해줘` → 적용
2. 적용 직후 우측 패널이 BNPL '소모품 구매' 행으로 자동 스크롤
3. 추가로 우측에서 그 행 데이터 (사용중 + 매월 1일 리셋) 확인

## 5. 알려진 제한 / 미구현

| 항목 | 상태 |
|---|---|
| 다음(Daum) 우편번호 검색 | 툴팁만 (실제 API 미연동) |
| 엑셀 템플릿 다운로드 / 실제 파싱 | 툴팁만, 업로드 시 더미 3명 고정 데이터로 대체 |
| 팀원 — 활성 멤버 (`/data/users`) 변경 | 채팅·모달은 **초대 대기** 만 추가, 활성 5명 리스트는 변경 안 됨 (다른 모듈 충돌 회피) |
| 승인 체계 / 적요 시나리오 | 채팅 시나리오는 있으나 store 미연결 → 우측 패널 시각 변화 없음 |
| BNPL 신청 승인 단계 | ⚙ 승인 처리 버튼 (프로토타입 전용 단축) |
| 한글 IME 이중 송신 (예전 버그) | 수정 완료 (조합 중 Enter 무시) |

## 6. 톤·UX 검증 포인트 (디자인 리뷰용)

- [ ] 채팅 입력 전: 추론 블록·시스템 칩 없음 (조용한 대화창)
- [ ] 채팅 입력 후: 추론 4단계 progressive 노출 (스피너 → 페이드인)
- [ ] 추론 등장 중 우측 자동 스크롤·자동 행 펼침 (펄스 색상은 의도적으로 제거됨)
- [ ] Diff 카드의 "편집 가능" 라벨 + 입력 박스 (인디고 톤)
- [ ] 적용 후: AI ack + 우측 자동 스크롤 (영향받은 행으로)
- [ ] 우측에서 직접 액션 시: 추론 없음, 패널 토스트 + 짧은 채팅 ack
- [ ] 모든 미구현 액션: PlannedTooltip 으로 "예정: …" 표시

## 7. 빠른 데모 스크립트 (5분)

1. **시작**: 우측 사이드바 하단 프로필 클릭 → 설정 진입 → 회사 설정 선택
2. `전체 설정 상태 알려줘` 입력 → 추론 등장 (기조 시연)
3. `마케팅팀 예산 500만원으로 올려줘` → 우측 패널 자동 전환 + 자동 펼침 시연
4. Diff 카드의 월 예산을 `7000000`으로 변경 → 미리보기 실시간 갱신 시연
5. 적용 → 우측 행 변경 + 자동 스크롤 시연
6. `분당 지사 배송지 추가해줘` → 인라인 편집 폼 시연 → 수령인·연락처 입력 → 적용
7. 우측 배송지 패널의 연필 아이콘 → 수정 모달 시연 → 저장 시 채팅 ack 발생
8. (선택) `김지현 결제권한 해제해줘` → Impact 카드 + 후속 suggestion 클릭 시연
