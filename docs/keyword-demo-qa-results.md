# 키워드 데모 QA 결과표

작성일:
- 2026-04-16

작성 기준:
- 코드/타입/정적 검토 기준
- `npx tsc --noEmit` 통과 확인
- `validateDemoScenarios()` 기준 시나리오 커버리지/필수 필드 검증 로직 반영 확인
- 브라우저 실기 클릭 테스트는 아직 별도 수행하지 않음

상태 정의:
- `코드기준 통과`: 코드/데이터/타입 기준으로 요구사항 충족이 확인됨
- `실기 확인 필요`: 실제 브라우저에서 직접 눌러보는 최종 확인이 남아 있음
- `이슈`: 코드 기준으로도 보완이 필요함

## 비고

- 현재 프로젝트에는 브라우저 기반 E2E 테스트 도구가 설정되어 있지 않아, 본 결과표는 정적 검증 중심이다.
- 다음 단계는 이 문서를 바탕으로 실제 수동 클릭 QA를 수행하고, 각 항목 상태를 `실기 확인 필요`에서 `통과` 또는 `이슈`로 갱신하는 것이다.

## 이슈 기록 원칙

- 이슈는 `재현 가능성`, `영향 범위`, `우선순위`가 바로 보이도록 짧게 적는다.
- 한 이슈에는 하나의 문제만 적는다.
- 키워드별 문제와 공통 회귀 문제를 섞지 않는다.
- 실제 브라우저에서 확인한 문제만 `재현 결과`에 적고, 추정 원인은 `추정 원인`에 분리해서 적는다.

## 이슈 상태 정의

| 상태 | 의미 |
|---|---|
| `new` | 새로 발견됨, 아직 분류 전 |
| `triaged` | 우선순위와 영향 범위를 확인함 |
| `fixed` | 코드 수정 완료, 재검증 대기 |
| `verified` | 재검증까지 완료 |
| `wontfix` | 데모 범위 밖이거나 이번 라운드 제외 |

## 이슈 우선순위 기준

| 우선순위 | 기준 |
|---|---|
| `P0` | 데모 진행 불가, 상세 패널 미오픈, 검색 흐름 붕괴 |
| `P1` | 핵심 시나리오 왜곡, 옵션 누락, 조건 카드/결과 카드 순서 오류 |
| `P2` | 문구, 레이아웃, 추천 이유 표현, 체감 품질 이슈 |
| `P3` | 사소한 텍스트, 간격, 비주얼 미세 조정 |

## 이슈 템플릿

```md
### [ISSUE-001] 모니터 검색 시 상세 패널에서 옵션 드롭다운이 보이지 않음

- 상태: `new`
- 우선순위: `P1`
- 키워드: `모니터`
- 범주: `상세 패널`
- 재현 환경: `local dev / Chrome`

#### 재현 절차
1. `모니터`로 검색한다.
2. 조건 카드를 제출한다.
3. 추천 결과에서 1순위 상품의 `상세 보기`를 누른다.

#### 기대 결과
- 우측 패널이 열리고 `옵션 선택` 드롭다운이 보여야 한다.

#### 실제 결과
- 우측 패널은 열리지만 옵션 선택 영역이 노출되지 않는다.

#### 영향 범위
- `모니터`
- 옵션 UI 공통 컴포넌트를 사용하는 다른 키워드 가능성 있음

#### 추정 원인
- 시나리오 상품 옵션 누락 또는 상세 패널 fallback 옵션 주입 경로 누락 가능성

#### 비고
- 최초 발견일: `2026-04-16`
- 관련 파일: `components/chat/ProductDetailPanel.tsx`, `data/chat-search-scenarios.ts`
```
