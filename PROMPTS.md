### 프롬프트 fix-2: 우측 GUI 아코디언 타임라인

```
우측 GUI 패널의 구매 플로우 표시 방식을 개선해줘.

=== 문제 ===
지금은 승인 → 결제 → 배송 각 단계가 전환되면 이전 단계 내용이 사라져서,
배송 단계에서 "이 주문이 어떻게 승인됐는지" 확인할 수 없어.

=== 해결: 아코디언 타임라인 ===
우측 패널(RightPanel)에 구매 플로우를 표시할 때,
각 단계를 별도 컴포넌트로 전환하지 말고 하나의 아코디언 타임라인으로 통합해줘.

components/commerce/OrderTimeline.tsx 를 새로 만들어:

  구조:
  ┌─────────────────────────┐
  │ ● 상품 선택        ✓ 완료 │  ← 접힌 상태 (클릭하면 펼침)
  ├─────────────────────────┤
  │ ● 품의 승인        ✓ 완료 │  ← 접힌 상태
  │   승인자: 김원균          │
  │   승인일: 2026-04-09     │
  ├─────────────────────────┤
  │ ● 결제             ✓ 완료 │  ← 접힌 상태
  ├─────────────────────────┤
  │ ◉ 배송 추적        진행중 │  ← 펼쳐진 상태 (현재 단계)
  │                          │
  │   접수 ──── 준비 ──── 배송중 │
  │   04/09   04/09    04/10  │
  │                          │
  │   택배사: CJ대한통운      │
  │   운송장: 1234-5678      │
  ├─────────────────────────┤
  │ ○ 구매확정          대기  │  ← 비활성 (아직 안 된 단계)
  └─────────────────────────┘

동작 규칙:
1. 현재 진행 중인 단계는 자동으로 펼쳐져 있음 (상세 내용 표시)
2. 완료된 단계는 접혀 있지만, 클릭하면 펼쳐서 상세 확인 가능
3. 아직 안 된 단계는 비활성 스타일 (회색, 클릭 불가)
4. 각 단계의 좌측에 타임라인 세로선 + 상태 원형 아이콘:
   - ✓ 완료: 초록 체크
   - ◉ 진행중: 파란 점 (pulse 애니메이션)
   - ○ 대기: 회색 빈 원

각 단계별 펼침 내용:
- 상품 선택: ProductRecommendCard 요약 (상품명, 가격, 수량)
- 품의 승인: 승인자, 승인일시, 자동/수동 여부, 승인 메모
- 결제: 결제수단, 결제금액, 결제일시
- 배송 추적: ShippingTracker 컴포넌트 (기존 것 재사용)
- 구매확정/반품: 구매확정 버튼 또는 반품 요청 폼

기존 컴포넌트 관계:
- ApprovalTracker, PaymentSelector, ShippingTracker는 독립 컴포넌트로 유지
- OrderTimeline이 이들을 각 단계의 펼침 영역에 임베드
- 단계 전환 시 OrderTimeline의 activeStep만 변경, 이전 컴포넌트는 읽기 전용으로 유지

데모:
- "다음 단계" 시뮬레이션 버튼은 패널 최하단에 배치
- 클릭할 때마다 activeStep이 다음으로 넘어가고, 이전 단계가 접힘
- 접힌 단계 클릭하면 다시 펼쳐서 내용 확인 가능
```
