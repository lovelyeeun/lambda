# Cockpit Prototype — 구매대행 AI 에이전트

## 프로젝트 개요
기업 구매대행 AI 에이전트 "cockpit"의 인터랙티브 프로토타입.
팀 공유 → MVP 범위 설정 + 개발 디테일 조율 용도.

## 기술 스택
- Next.js 16 + TypeScript + Tailwind CSS v4
- 백엔드 없이 프론트엔드 목업
- 더미 데이터는 `/data/` 폴더에 중앙 관리
- 디자인 시스템: DESIGN.md 참조 (ElevenLabs 인스파이어드)

## 핵심 설계 원칙

### 레이아웃
- **3단 레이아웃**: 좌 사이드바(240px) / 중앙(flex) / 우측 GUI(360px, 기본 접힘)
- **탭 전환**: {회사명} ↔ 스토어 — 같은 셸, 사이드바+중앙 내용만 교체 (Claude Code의 채팅-Cowork-코드 탭 전환 방식)
- **우측 GUI**: 필요시 펼침/접힘 (Claude 사이드바 방식). 장바구니, 승인상태, 배송추적, 상품상세 등 컨텍스트별 내용 교체
- **설정**: 전체 오버레이 모달 (Notion 설정 방식). 좌측 하단 프로필 아이콘 클릭 → 오버레이로 개인설정/회사설정 진입

### 채팅
- 하나의 채팅에서 여러 주제 다룰 수 있음
- 에이전트(@주문, @배송, @권한)는 내부 자동 감지 — 의도에 따라 자동 호출, 별도 멘션 불필요
- 결제 완료 후 채팅 유지 (닫히지 않음) + 채팅 리스트에 "완료" 뱃지 표시
- 온보딩 채팅: 같은 UI + 특별 배경색으로 일회성 모드 인지

### 구매 플로우
- 품의 승인 UI는 하나, 진입 경로만 다름 (채팅/사이드바/GUI)
- 자동승인(소액/규정) vs 담당자 알림(고액/예외) 분기
- 직접구매: 관리자가 부여한 권한이 있는 경우 품의 생략하고 바로 결제
- 결제수단: 등록 후 자동 적용, 복수일 때 선택. 최초 시 등록 유도
- 장바구니: 우측 패널에서 열림

### 스토어
- 상품 리스트 → 상세페이지 있음
- "회사 상품폴더에 담기" → {회사명} 탭의 좌측 사이드바에만 반영 (스토어에는 회사 상품폴더 미노출)
- 앱(렌탈/화환/주문제작): 설정에서 on 시에만 스토어 사이드바에 노출. 현재는 "준비중" 수준
- 스토어→채팅 전환: 탭 자체가 {회사명}으로 전환됨 (토스트/트랜지션 추가)
- SCM: CRM 테이블+상세, RFQ는 채팅+우측 GUI, 계약 서명까지 서비스 내 진행

### 채팅 리스트 전체 보기 (C-9)
- 사이드바 미노출, 중앙 영역 전체를 노션 DB 스타일 뷰로 사용
- 미리 정해진 뷰 전환 (날짜/조직/태그) + 새 보기 추가 가능

### 비용 인텔리전스
- 3단 레이아웃 유지 (중앙 채팅에서 분석, 우측 GUI에서 디테일/내보내기)
- 외부 지출 데이터 업로드(엑셀 등) 가능
- 내보내기: Excel, PDF, PNG, 구글드라이브

### 설정 (오버레이)
- 좌측 하단 프로필 아이콘 클릭 → 전체 오버레이 모달
- 좌측: 설정 메뉴 (개인설정/회사설정/앱관리/알림)
- 우측: 설정 콘텐츠
- 회사 지식 추가: 프롬프트 입력 + 데이터 업로드 + 태그 편집 (간단한 UI)
- 예산: 설정에서만 관리 (채팅으로 수정 요청 시 확인 후 반영)
- 과금: 작업 단위 추상화 (내부 토큰+주문 혼합, 유저에게는 사용량 표시)

## 인터랙션 규칙
- **모든 클릭 가능 요소가 작동해야 함** — 디자인 미흡은 OK, 전환 안 되는 건 안 됨
- 미구현 액션은 `<Tooltip>` 컴포넌트로 "예정: [설명]" 표시
- 모든 클릭 가능 요소에 `cursor-pointer` + hover 효과
- 화면 간 전환은 Next.js 라우팅 사용
- 모달/오버레이: 설정, 온보딩 CTA
- 반응형: 데스크탑 우선, 태블릿 최소 대응, 모바일 전용 없음

## 작업 규칙
- **작업 시작 전 PROGRESS.md를 먼저 읽고 현재 진행 상태와 알려진 이슈를 파악할 것**
- 한 번에 한 모듈씩 작업
- DESIGN.md의 디자인 토큰을 반드시 따를 것 (색상, 타이포, 섀도, 버튼 스타일)
- 더미 데이터는 `/data/` 폴더에 중앙 관리, 하드코딩 금지
- 컴포넌트는 재사용 가능하게 분리
- 모든 페이지에서 탭 전환({회사명} ↔ 스토어)과 설정 진입이 작동해야 함

## 공유 컴포넌트 (10개)
1. AppShell (상단바 + 모드 탭 + 프로필)
2. ChatUI (채팅 버블, 입력창, AI 응답 카드)
3. RightPanel (접힘/펼침, 컨텍스트별 내용 교체)
4. ApprovalTracker (요청→대기→승인/반려)

9. OnboardingCTA (모달/CTA 버튼, 최초 1회)
10. ExportMenu (Excel, PDF, PNG, 구글 드라이브)
